# 다음 마일스톤 시작

현재 마일스톤이 완료됐는지 확인하고 다음 마일스톤 계획을 세웁니다.

## 실행 순서

1. **현재 마일스톤 식별**
   - 사용자에게 "현재 M1~M5 중 어느 단계까지 완료했나요?" 질문
   - 또는 최근 커밋 이력에서 추론

2. **완료 정의(DoD) 검증**
   - `SPEC.md`의 "완료 정의" 섹션 확인
   - 체크리스트 하나씩 검증:
     - [ ] TypeScript 오류 0개
     - [ ] ESLint 경고 0개
     - [ ] `pnpm build` 성공
     - [ ] 모바일/PC 양쪽 확인됨
     - [ ] Lighthouse 90+ (해당 시)
     - [ ] 빈 상태/로딩/에러 상태 처리됨
     - [ ] 관련 문서 업데이트됨

3. **빠진 항목 리포트**
   - DoD 중 충족 안 된 항목 나열
   - 각각 해결 방법 제시
   - 사용자가 "마저 완료하자" 또는 "다음으로 넘어가자" 선택

4. **다음 마일스톤 계획**
   - 다음 마일스톤의 핵심 기능 요약 (SPEC.md 기반)
   - 추가로 필요한 라이브러리 리스트
   - 예상 작업 단위 분할 (커밋 몇 개로 나눌지)
   - DB 스키마 변경 필요한지

5. **컨텍스트 파일 업데이트 제안**
   - 만약 새 규칙이 생겼다면 CLAUDE.md에 추가 제안
   - 새 컴포넌트가 필요하면 공통 컴포넌트 목록에 추가 제안

## 주의

- DoD가 하나라도 충족 안 되면 **다음 마일스톤 시작 금지**. "마무리부터 하자"고 안내.
- 사용자가 강행하고 싶어하면 **남은 TODO를 PROGRESS.md에 기록**해둠.
