# 코드 쉽게 설명하기

사용자는 비개발자입니다. 방금 작성한 코드나 지정한 파일을 **한글로, 비유를 써서** 설명해주세요.

## 설명 원칙

1. **전문 용어는 한 번 쓰고 옆에 괄호로 풀어 설명**
   - 예: "useState (리액트에서 값을 기억하는 기능)"

2. **비유 사용**
   - `useState` → "포스트잇에 메모하는 것"
   - `useEffect` → "특정 사건이 일어날 때마다 알림 받기"
   - `props` → "부모가 자식에게 도시락 싸주는 것"
   - API 호출 → "우체국에 편지 보내고 답장 기다리기"

3. **"왜?"를 먼저**
   - "이걸 왜 쓰는지"를 먼저 설명하고, "어떻게 쓰는지"를 나중에

4. **그림 그리기**
   - 데이터 흐름을 화살표로 표현 (텍스트 아트 OK)
   - 예:
     ```
     사용자 클릭 → API 호출 → DB 저장 → 성공 메시지 표시
     ```

5. **이 코드가 UI에서 어디에 보이는지 알려주기**
   - "이 함수는 오른쪽 미리보기 카드 상단 이름을 그리는 코드입니다"

## 출력 형식

```
📝 이 파일이 하는 일:
(한 줄 요약)

🎯 왜 필요한지:
(간단한 이유)

🔍 주요 부분:
1. 첫 번째 블록: ...
2. 두 번째 블록: ...

💡 비유로 이해하기:
...

🔗 관련 파일:
- ...
```

## 사용자가 추가로 물어볼 만한 질문 준비

설명 후 이렇게 덧붙이세요:
> "더 궁금한 부분 있으세요? 예를 들어:
> - 이 코드 없으면 어떻게 돼?
> - 여기 숫자 바꾸면 뭐가 달라져?
> - 다른 곳에서 똑같이 쓸 수 있어?"
